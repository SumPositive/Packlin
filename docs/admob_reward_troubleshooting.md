# AdMob リワード広告がコンソールに出ないときの確認事項

## 症状
- AdMob コンソールのレポートに報酬型広告ユニット `reward_1` が表示されない / インプレッションが 0 のまま
- アプリ側では広告を視聴しようとしているが、そもそも広告配信が始まらない

## 主な原因候補

### 1. デバッグビルドではテスト用ユニット ID を使っている
- `#if DEBUG` ブロックで、リワード広告 ID が公式のテスト用 ID に差し替わっているため、実機であっても広告リクエストがテスト扱いになる。
- テスト ID へのアクセスは AdMob コンソールの収益レポートには計上されないので、`reward_1` が一覧に出てこない。
- 本番の広告ユニットを使うのは Release ビルドだけなので、コンソール確認時は Release 配布（TestFlight / App Store）で試すこと。

### 2. 広告シートを開かないとリワード広告を読み込まない
- 報酬型広告のローダーは `AdMobAdSheetView` が表示されたタイミングで生成・読み込みが走る。
- ユーザーが広告シートを一度も開いていない場合、リワード広告のリクエスト自体が発生しないためコンソールにも統計が反映されない。
- 実機で再現する際は、必ずアプリ内の「広告を見て寄付」シートを開いてリクエストが発生する状態を作る。

### 3. GoogleMobileAds がリンクされない環境ではプレースホルダーに置き換わる
- AdMob SDK がリンクできないビルドターゲット（プレビューや一部のテスト環境）では `#if canImport(GoogleMobileAds)` の分岐でプレースホルダー表示に切り替わる。
- この場合も広告リクエストは送信されないため、コンソールには何も表示されない。

## チェックリスト
- 試しているビルドが Release（本番ユニット ID を使う）か確認する。
- アプリ内で「広告を見て寄付」シートを開き、動画広告の読み込み・再生を実行する。
- アプリのバージョンが AdMob SDK を正しくリンクできるターゲットでビルドされているかを確認する。

## TestFlight でバナーも `reward_1` も出ないときの追加確認
- バナーは広告の読み込みに失敗すると UI 上に理由と再読み込みボタンが表示される。TestFlight 端末で再読み込みを押し、エラーメッセージが変化するか確認する。
- 報酬型広告が Error 表示になった場合、Crashlytics にも詳細エラーを自動送信する実装にしてあるため、同時刻のログを確認する（フィルレート不足か、構成ミスかを切り分けやすい）。
- TestFlight 端末で App Tracking Transparency の許可ダイアログを拒否していても広告は配信されるが、ネットワーク到達性が悪い場合は空配信となる。Wi-Fi・モバイルデータが有効かも併せて確認する。
